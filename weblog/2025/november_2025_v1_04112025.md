This example models a daily temple supply chain for offerings (flowers, milk, ghee, grains) from multiple vendors to multiple rituals, with time windows and purity levels. It demonstrates AMPLâ€™s separation of model, data, and control.

set ITEMS;
set RITUALS;
set TIMES ordered;  # discrete slots within the ritual day

param demand{RITUALS, ITEMS} >= 0;           # units required by each ritual
param supply{VENDORS, ITEMS} >= 0;           # max available units at vendors
param cost{VENDORS, ITEMS} >= 0;             # cost per unit from vendor
param purity{VENDORS, ITEMS} >= 0;           # purity score (e.g., 0â€“1)
param min_purity{RITUALS, ITEMS} >= 0;       # minimum purity per ritual-item
param travel_time{VENDORS, RITUALS} >= 0;    # time slots needed to deliver
param ritual_slot{RITUALS} in TIMES;         # time slot when ritual occurs

Decision variables
var x{VENDORS, RITUALS, ITEMS} >= 0;         # quantity shipped vendorâ†’ritual
var y{VENDORS, RITUALS, ITEMS} binary;       # 1 if any quantity shipped

Objective: minimize total procurement cost
minimize TotalCost:
    sum{v in VENDORS, r in RITUALS, i in ITEMS} cost[v,i] * x[v,r,i];

Demand fulfillment
s.t. DemandSatisfied{r in RITUALS, i in ITEMS}:
    sum{v in VENDORS} x[v,r,i] >= demand[r,i];

Vendor capacity
s.t. VendorCapacity{v in VENDORS, i in ITEMS}:
    sum{r in RITUALS} x[v,r,i] <= supply[v,i];

Purity constraints: any shipped quantity must meet ritualâ€™s minimum purity

    purity[v,i]  y[v,r,i] >= min_purity[r,i]  y[v,r,i];

param M >= 0 default 1e6;
s.t. LinkXY{v in VENDORS, r in RITUALS, i in ITEMS}:
    x[v,r,i] <= M * y[v,r,i];

    z[r,i] >= sum{v in VENDORS} y[v,r,i] - 1;

Ritual scheduling: multi-ritual calendar with time, priest allocation, and sanctity windows

This model schedules a set of rituals across a day with fixed time windows, resource (priest) constraints, precedence, and purification buffers. It captures the worldview that constraints encode sanctity and flow.

---

Model

```ampl

ritual_scheduling.mod

set RITUALS ordered;
set PRIESTS;
set SLOTS ordered;

Parameters
param duration{RITUALS} >= 1;                # number of slots required
param window_start{RITUALS} in SLOTS;
param window_end{RITUALS} in SLOTS;
param requires_pure{RITUALS} binary;         # 1 if needs purification buffer
param buffer >= 0 default 1;                 # slots of buffer before start
param precedence{RITUALS, RITUALS} binary default 0;  # precedence matrix
param priest_skill{PRIESTS, RITUALS} binary; # 1 if priest is qualified
param priest_capacity{PRIESTS} >= 0;         # max rituals per day per priest

Decision variables
var assign{RITUALS, SLOTS} binary;           # ritual r starts at slot s
var use{PRIESTS, RITUALS} binary;            # priest p assigned to ritual r

Objective: maximize weighted sanctity and on-time performance
param weight{RITUALS} >= 0 default 1;
maximize SanctityScore:
    sum{r in RITUALS, s in SLOTS} weight[r]*assign[r,s];

Time window constraints
s.t. WithinWindow{r in RITUALS}:
    sum{s in SLOTS} assign[r,s] = 1;

s.t. RespectWindow{r in RITUALS}:
    sum{s in SLOTS: s < windowstart[r] or s > windowend[r]} assign[r,s] = 0;

Non-overlap in slots: build occupancy
set OCC{r in RITUALS, s in SLOTS} :=
    {t in SLOTS: ord(t) >= ord(s) and ord(t) < ord(s)+duration[r]};

var occupied{RITUALS, SLOTS} binary;
s.t. Occupy{r in RITUALS, s in SLOTS}:
    occupied[r,s] >= assign[r,s];

At most one ritual per slot (if single sanctum)
s.t. SingleSanctum{s in SLOTS}:
    sum{r in RITUALS} occupied[r,s] <= 1;

Purification buffers: forbid starts that violate buffer
s.t. PurityBuffer{r in RITUALS}:
    if requires_pure[r] = 1 then
        sum{s in SLOTS: ord(s) < ord(windowstart[r]) and ord(windowstart[r]) - ord(s) < buffer}
            assign[r,s] = 0;

Precedence constraints: if precedence[a,b]=1 then start(a)+dur(a) <= start(b)
var start_idx{RITUALS} integer >= 1;
s.t. StartIndexDef{r in RITUALS}:
    start_idx[r] = sum{s in SLOTS} ord(s)*assign[r,s];

s.t. PrecedenceRule{a in RITUALS, b in RITUALS: precedence[a,b] = 1}:
    startidx[a] + duration[a] <= startidx[b];

Priest qualification and capacity
s.t. PriestQualified{p in PRIESTS, r in RITUALS}:
    use[p,r] <= priest_skill[p,r];

s.t. PriestPerRitual{r in RITUALS}:
    sum{p in PRIESTS} use[p,r] >= 1;

s.t. PriestCapacity{s in SLOTS, p in PRIESTS}:
    sum{r in RITUALS} occupied[r,s] * use[p,r] <= 1;

s.t. DailyCapacity{p in PRIESTS}:
    sum{r in RITUALS} use[p,r] <= priest_capacity[p];
```

# Comprehensive Explanation of Foo Dogs: Symbolism and Historical Evolution

## Introduction and Origins

- **Early Records and Introduction to China**

  - The Chinese word *Shi* is believed to derive from the Persian word *Ser*, meaning lion.  
  - Han imperial records mention lions being brought to the Han court by emissaries from Central Asia and Persia.  
  - These lions became symbols of protection and were used to adorn tombs and shrines with stone and earthen sculptures.

- **Early Examples**  
  - The lion statues of Gaoyi Que in Hunan, dating from the Han Dynasty, are among the earliest known pairs of lion sculptures guarding ceremonial gate pillars.
